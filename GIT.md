# Работа с удаленным репозиторием
## Перенос файлов с GitHub в локальный ропозиторий
1. **На GitHub**
```sh
открыть нужный репозиторий -> code (верхний правый угол) -> копируем ссылку HTTPS
```
2. **В VScode**
```sh
открываем папк в которую будем размещать репозиторий
```
```sh
git clone <ссылка репозитория>
```
### 3. Появляется **локальный репозиторий**

- -> в нашей папке **создается папка**

(название папки - название проекта на GitHub)

При вводе команды "git status" - система будет говорить что **мы не "init"**

-> это происходит из-за того что **мы находимся в созданной нами папки**, а не в папке перенесенной из интернета

в нашей папке - нет репозитория

в пренесенной - есть репозиторий

- -> нужно **провалиться дальше в папку**
```sh
cd <name_папки>
```
выдача терминала должна измениться

_**выйти из папки в предыдущую**_
```sh
cd ..
```

4. **Можно работать с файлами!**

## Конфликт (rejected)
**между локальным репозиторием и удаленным сервиром** после "push" 

-сначала скачиваем все изминения **с сервера** на локальный репозиторий, где выбираем нужную нам инфо
```sh
git pull --rebase
```
-add и commit

-затем
```sh
git rebase --continue
```
-и отправляем изминения на сервер
```sh
git push
```

# < Pull requeste > - Запрос вливания своих изминений в чей-то репозиторий
для участия в проекте необходимо:
- зайти на GitHub в нужный репозиторий
- сделать себе  на аккаунтGitHub копию репозитория 
```sh
кнопка "Fork"(сверху справа)
```
- далее клонируем сохраненный репозиторий в наш локальный репозитарий на компьютере

См. п. "Перенос с GitHub в локальный ропозиторий"

- в VScode **создаем отдельную ветку NEW**


**! Все изминения в чужом проекте всегда делаются в своей созданной ветки !**

```sh
git branch <name_new_branch>
->
git checkout <name_new_branch>
```
- **создаем новый файл "README.md"**

- **Теперь можем работать над проектом**

### - Отправка изминений на GitHub
c NEW branch 
```sh
git push...
```

**на GitHub:**
- **появляется новая ветка** созданная нами в локальном репозитории

*в случае слияния веток и удаления второстепенной в терминале ->push, на GitHub второстепенная ветка автоматически не удалитmся*

**удаление второстепенной ветки на GitHub** через команду терминала
```sh
git push origin --delete <name_branch>
```

- **появляется кнопка "Compara & pull requeste"**
с помощью которой моем отапрвить хозяину репозитори предложенные изминения
- **"Create pull request"** - отправляем
